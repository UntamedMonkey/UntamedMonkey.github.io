1.工程结构

  • 保证仓库结构清晰、易读，如示例1.1所示。
  • 代码间低耦合，无循环依赖，少使用全局变量，少写长函数
  • 拆分出文件（模块）、文件夹（包）；合理使用相对import和绝对import
  • 合理使用面向对象、装饰器、with(上下文管理器；示例1.2)等特性，有利于代码解耦。

2.代码风格

  • 在存在各种黑魔法的Python中，我们提倡最明确和直接的编码方式
  • 在同一行代码中写 两条独立的语句是糟糕的。
  • 少使用*args定义函数参数,因为它不应该被用在能用更简单和更明确的结构，来足够表达函数意图的情况中。
  • 任何不开放给客户端代码使用的方法或属性，应该有一个下划线前缀。
  • 函数只有一个出口点，输入、运行环境的检查建议使用异常机制，见例2.1
  • 可能运行失败的代码使用异常机制（try...except）
  • 使用上下文管理器管理资源
  • 使用属性，而不是get/set
  • 使用_命名临时变量
  • 避免对不同类型的对象使用同一个变量名
  • string类型是不可变类型，减少内存分配，少使用s+=str(i)，可以改成：生成list，再join。
  • 使用 str.format() 替代 % 操作符。例如：foobar = '{foo}{bar}'.format(foo=foo, bar=bar)
  • 使用in进行字串匹配
  • 使用enumerate()进行index计数
  • 使用next()进行按条件查找
  • 对于可变序列，使用del替代重新分配del l[j:] instead of l = l[:j]
  • 如果您知道一个列表或者元组的长度，您可以将其解包并为它的元素取名。a, b, *middle, c = [1, 2, 3, 4]
  • 使用filter() 函数过滤list：sequence[::] = list(filter(lambda i: i != x, sequence))
  • 修改list，优先考虑赋值为新的list，如：a = [i + 3 for i in a] or a = list(map(lambda i: i + 3, a))
  • 过长的行一个更好的解决方案是在周围使用括号 (·····)
